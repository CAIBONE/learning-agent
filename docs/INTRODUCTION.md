# 学吧 — AI 智能学习助手

## 一、作品定位

「学吧」是基于 OpenClaw 平台打造的 AI 智能学习助手，通过飞书渠道提供完整学习闭环：

**目标拆解 → 知识图谱 → 计划排程 → 教材生成 → 独立审计 → 内容推送 → 测验评估 → 独立审计 → 动态调整 → 复盘报表**

无论是备考拿证、掌握一项技能、还是流利说一门外语，学吧都能将模糊的学习需求转化为可执行的知识图谱和学习路径，通过飞书主动推送学习内容，定期测验检验成果，并根据学习数据动态调整计划，实现个性化学习。

## 二、目标类型自适应

学吧根据学习目标是否有外部验收标准，走两条不同路径：

| 类型         | 特征                                 | 示例                        | 核心流程                                                     |
| ------------ | ------------------------------------ | --------------------------- | ------------------------------------------------------------ |
| **有标准型** | 有考试大纲/等级框架/证书要求         | 考中级经济师、IELTS 7.0     | 获取官方大纲 → 按考纲生成知识树 → 以分数/等级为验收目标      |
| **无标准型** | 没有外部验收标准，"学会"需要自己定义 | 掌握 LLM 原理、流利英语口语 | 了解使用场景 → Agent 引导定义里程碑 → 知识树 + 里程碑并行验收 |

**无标准型的核心创新 — 项目引导 + 自适应验收：**

当目标属于项目驱动型（编程/技术类），用户通常不知道做什么项目。学吧会：

1. 了解用户的工作场景和兴趣方向
2. 联网检索该领域的入门级项目清单
3. 推荐 3-5 个具体项目（标注难度和预计时间），**不是泛泛的"做个 Web 应用"，而是"做一个命令行记账工具：SQLite 存储，月度统计报表"**
4. 用户选择后定义明确的交付物标准
5. 项目可中途替换，已有知识不浪费

根据科目特性，学吧建议不同的里程碑风格：

| 科目特性         | 里程碑风格 | 验收方式                       |
| ---------------- | ---------- | ------------------------------ |
| 编程/技术类      | 项目驱动   | 代码运行 + 实操演示 + 代码审查 |
| 语言类（不考证） | 场景驱动   | 角色扮演对话 + 口语录音评估    |
| 设计/艺术类      | 作品驱动   | 作品提交 + Agent 评估          |
| 理论/知识类      | 输出驱动   | 口头讲解 + Agent 提问检验      |
| 工具/软件类      | 任务驱动   | 实操演示 + 独立完成工作流      |

## 三、完整工作流

### 阶段 1：目标拆解与路径规划

**触发**：用户说"我想学 XXX"

1. **目标类型判断**：问用户"学了之后怎么算学会？有没有考试/证书/等级标准？"
2. **有标准型**：联网获取考纲/等级框架 → 按大纲生成知识树
3. **无标准型**：了解使用场景 → 联网检索能力模型 → 引导定义里程碑 → 知识树与里程碑并行
4. **飞书权限一次性预检**：首次创建学习项目时，一次性申请全部飞书权限（180+ scope，涵盖文档/多维表格/知识库/云空间/消息/通讯录/日历/任务等）
5. **6 步语义验证**：Schema 验证 → nodeId 唯一性 → prerequisites 引用完整性 → 循环依赖检测 → 深度/时长合理性 → 内容合理性抽检
6. **独立审计 Agent 审查**：覆盖率 ≥ 70%、学时可行性检查、session-notes 合规性检查（最多重试 3 次，仍不通过则提交用户裁决）
7. **自动创建飞书目录**：在飞书知识库自动创建学科文件夹 + 章节子文件夹 + 复习资料/报表目录

### 阶段 2：学习计划排程

**排程算法**：

- 基于知识树前置依赖关系拓扑排序
- 可用天数 × 每日时长 - 20% 缓冲 = 可用学时
- 艾宾浩斯遗忘曲线自动插入复习节点（1→3→7→14→30 天）
- 每个模块结束设阶段测试里程碑
- 无标准型目标同步设置里程碑验收节点

### 阶段 3：内容生成与推送

1. **联网检索**：搜索该知识点权威资料（中文 + 英文 + 考点 + 案例）
2. **深度教材生成**：不是搜索结果拼接，而是基于检索素材原创的完整教材（2000-5000+ 字，10 个必需小节）
3. **写入 Session Notes**：将对话中产生的用户需求、偏好、纠错等信息持久化，供审计 Agent 和后续会话使用
4. **独立审计 Agent 审查**：字数达标 + 结构完整 + 知识准确性交叉验证 + 前后一致性 + 练习题可解性 + 自适应策略匹配 + session-notes 合规性（硬指标必须修复，软指标可带标记推送）
5. **飞书文档推送**：写入飞书文档 → 消息卡片推送链接 → 交互按钮（已完成/有疑问）
6. **学习速度自适应**：学得快 → 推送更多更深内容；学得慢 → 精简内容降低难度

### 阶段 4：测验评估

**5 种测验类型**：随堂测验（1-3题）/ 每日回顾（3-5题）/ 每周回顾（5-10题）/ 月度评估（10-20题）/ 考前模拟（30-50题）

**题量评估算法**：`最终题量 = baseCount × difficultyMultiplier × importanceMultiplier × fatigueAdjustment`

**独立审计 Agent 审查题目**：题型比例 + 难度分布 + 答案正确性重解验证 + 去重率 < 20% + 选项质量检查 + 错题覆盖 + session-notes 合规性

**里程碑验收**（无标准型）：到达里程碑节点时，Agent 通过实际检验确认达标——口头讲解/实操演示/角色扮演/场景模拟/作品提交/录音录像

### 阶段 5：动态调整

**三大量化指标**：

- `efficiencyRatio` = 预估时长 / 实际时长（>1.2 学得快，<0.8 学得慢）
- `masteryGainRate` = 掌握度提升 / 测验次数（>0.3 提升快，<0.1 提升慢）
- `scheduleVariance` = 已完成 / 计划数（>1.1 超前，<0.9 落后）

**分档策略**：

| 掌握度 | 策略                         |
| ------ | ---------------------------- |
| ≥ 80%  | 跳过复习，后续加速 ×0.8      |
| 60-79% | 按原计划继续                 |
| 40-59% | +1 次专项练习，后续减速 ×1.3 |
| < 40%  | 暂停新内容，重新讲解         |

### 阶段 6：可视化报表与复盘

- **飞书多维表格 Dashboard**：折线图（掌握度趋势）、柱线组合（测验成绩）、饼图（错题分布/知识完成度）、看板（知识图谱）+ 7 个命名视图 + 1 个仪表盘
- **学习复盘**：数据→洞察→行动闭环，回答"为什么这样"和"接下来怎么办"
- **考前冲刺模式**：距考试/里程碑 ≤ 14 天自动切换，聚焦高频考点，每日 3 次推送

## 四、技术架构

### 多 Agent 架构

```
┌─────────────────────────────────────────────────┐
│              学吧 Main Agent                      │
│  上下文：SOUL + 路由 + 对话历史 + 学生档案          │
│                                                   │
│  内置 Skill：goals / knowledge-tree / plan /      │
│              content / quiz / review / reports /   │
│              cron / feishu-sync                    │
│                                                   │
│  生成完成 → 写 session-notes → 派发 Audit Agent    │
└──────────────────────┬──────────────────────────┘
                       │
                       ▼
         ┌──────────────────────────┐
         │   Audit Agent（独立上下文）│
         │   技能：learning-audit    │
         │   看不到生成推理过程       │
         │   自主读取数据文件审计      │
         └──────────────────────────┘
```

**为什么拆出独立审计 Agent**：既当运动员又当裁判会出问题。Audit Agent 在隔离上下文中运行，只看到生成物、数据文件和 session-notes，确保审计客观性。

**Session Notes 机制**：对话中产生的关键信息（用户偏好、纠错、学习状态变化）持久化到 `session-notes.yaml`，解决"对话上下文不共享"问题，供 Audit Agent 审计验证和后续会话恢复个性化策略。

### 审计失败处理流程

```
审计不通过 → Main Agent 按 fixAction 修复 → 重新派发审计
                                               ↓
                                     最多重试 3 次
                                               ↓
                             仍不通过 → 提交用户裁决
```

- **硬指标（hard）**：字数/结构/答案正确性等客观问题，必须修复才能推送
- **软指标（soft）**：知识准确性偏差等主观问题，可带标记推送
- **用户裁决选项**：接受（带标记推送）/ 按审计建议修改 / 重新生成 / 跳过

### 数据持久化

- **本地存储（primary）**：YAML/JSON/JSONL 文件，按 studentId 隔离
- **飞书同步（secondary）**：知识库 + 多维表格 5 张数据表 + 7 个视图 + 1 个仪表盘，双向同步
- **映射表**：`feishu-mapping.yaml`（飞书 token/视图/仪表盘映射）+ `mapping.yaml`（openid→studentId 映射）
- **对话笔记**：`session-notes.yaml`（对话衍生信息持久化，供审计和上下文恢复）

## 五、飞书深度集成

| 集成项   | 说明                                                         |
| -------- | ------------------------------------------------------------ |
| 知识库   | 自动创建学科目录 + 章节子文件夹 + 复习资料/报表目录          |
| 文档     | 学习内容写入飞书文档，消息卡片推送链接                       |
| 多维表格 | 5 张数据表（知识节点/学习记录/测验记录/错题本/掌握度追踪）+ 7 个命名视图 + 1 个 Dashboard |
| 消息     | 学习内容/测验/报表通过消息卡片推送，含交互按钮               |

## 六、多用户支持

- 每个用户通过飞书 open_id 自动识别，映射到唯一 studentId
- 定时任务按学生隔离：`learning-<type>-<studentId>-<subjectId>`
- 数据完全隔离，互不干扰
- 支持多科目并行学习

## 七、部署方式

```bash
git clone https://github.com/CAIBONE/learning-agent.git
cd learning-agent
bash setup.sh
```

AI 工具自动部署详见 `docs/DEPLOY-PROMPT.md`，将提示词复制到 Openclaw / Hermes Agent 聊天框即可自主完成部署。

## 八、创新点

1. **项目引导机制**：项目驱动型目标中，Agent 主动根据用户工作场景联网推荐具体项目，定义交付物标准
2. **独立审计 Agent**：生成和审计在隔离上下文中运行，避免"既当运动员又当裁判"。审计 Agent 看不到生成推理过程，只基于生成物和数据文件独立判断
3. **Session Notes 机制**：对话中产生的用户需求、纠错、学习状态变化持久化到文件，解决多 Agent 架构下的上下文传递问题，同时支持后续会话恢复个性化策略
4. **审计失败用户裁决**：硬指标必须修复，软指标可带标记推送。重试 3 次仍不通过时提交用户裁决，用户可选择接受/修改/重新生成/跳过
5. **量化动态调整**：三个量化指标（效率比/掌握度提升率/进度偏差）驱动计划调整
6. **题量评估算法**：动态决定每次测验题量，考虑难度、重要性、疲劳度
7. **双轨冲刺模式**：考试型 deadline ≤ 14 天激进冲刺，无标准型里程碑到期 ≤ 7 天温和冲刺
8. **可视化报表**：飞书多维表格内置图表（折线图/柱状图/饼图/看板）生成 Dashboard，7 个命名视图 + 1 个仪表盘
