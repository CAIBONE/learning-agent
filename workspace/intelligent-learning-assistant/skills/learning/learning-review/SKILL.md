# Learning Review — 学习复盘

## 定位

独立的学习复盘模块。不依赖报表生成的可视化，专注于**"数据→洞察→行动"**的闭环。
报表回答"现在怎样"，复盘回答"为什么这样"和"接下来怎么办"。

## 触发条件

- 用户主动要求"帮我复盘"、"学习情况怎么样"、"总结一下"
- 阶段测验完成后自动触发
- 每周复盘 cron（周五 18:00）
- 每月复盘 cron（每月 1 号 10:00）
- 学习目标完成/到期时触发

## 数据路径

- 掌握度：`data/<studentId>/mastery.json`
- 测验日志：`data/<studentId>/quiz-results.jsonl`
- 内容日志：`data/<studentId>/content-log.jsonl`
- 错题本：`data/<studentId>/wrong-answers.jsonl`
- 计划：`data/<studentId>/plans/<subjectId>.yaml`
- 目标：`data/<studentId>/goals.yaml`
- 知识树：`data/<studentId>/<subjectId>.yaml`
- 飞书多维表格：`feishu_bitable`（如已同步）
- 复盘输出：`data/<studentId>/reviews/<period>.md`

## 复盘流程

### 第 1 步：数据汇总

读取所有相关数据，计算以下指标：

**进度指标：**
- 已完成节点数 / 总节点数 = 完成率
- 实际学习天数 / 预期学习天数 = 出勤率（预期天数 = 用户声明的每周学习天数 × 周数）
- 实际学习时长 vs 计划时长 = 时长达成率
- 连续学习天数
- **里程碑进度**（无标准型）：已达标里程碑数 / 总里程碑数

**掌握度指标：**
- 各模块平均掌握度
- 掌握度 ≥ 80% 的节点占比（熟练率）
- 掌握度 < 40% 的节点占比（薄弱率）
- 掌握度变化趋势（上升/稳定/下降）

**测验指标：**
- 测验总次数、总题数、总正确率
- 各题型正确率（根据目标类型题目不同）
- 最近 3 次测验成绩趋势
- 错题总数、各错误类型占比
- **里程碑验收结果**（无标准型）：各里程碑 pass/fail 记录

**效率指标：**
- 平均每个节点的学习时长
- 平均内容完成时间 vs 预估时间 = 效率比
- 内容-掌握度转化率（每篇内容带来的掌握度提升）

### 第 2 步：洞察分析

基于数据，回答以下问题：

**1. 哪些学得好？为什么？**
- 找出掌握度提升最快的 3 个节点
- 分析共同特征（类型？难度？学习方式？）

**2. 哪些学得差？为什么？**
- 找出掌握度最低/下降的 3 个节点
- 分析错误类型分布
- 判断原因：内容难度？基础不足？复习不够？

**3. 学习效率如何？**
- 效率比 > 1.2 → 学得快，可以增加难度
- 效率比 0.8-1.2 → 正常节奏
- 效率比 < 0.8 → 学得慢，需要调整

**4. 计划执行如何？**
- 完成率 > 90% → 计划合理，执行力强
- 完成率 70-90% → 基本达标，有小缺口
- 完成率 < 70% → 计划可能不合理或执行力不足

**4+. 里程碑进度（无标准型）：**
- 已达标里程碑 / 总里程碑 → 整体进度
- 当前里程碑是否在 targetDate 前完成 → 时间管理评估
- 以当前节奏，下一个里程碑预计 X 周后可达标 → 预测性提醒

### 第 3 步：生成复盘报告

**日报格式（简洁）：**
```
📋 学习日报 — YYYY-MM-DD

今日完成：X 个节点，Y 分钟
测验成绩：Z/W（正确率）
掌握度变化：+N 个节点提升

明日计划：[节点列表]
⚠️ 需关注：[薄弱点]
```

**周报格式（完整）：**
```
📊 学习周报 — YYYY-MM-DD ~ YYYY-MM-DD

═══ 本周概览 ═══
完成率：X/Y（Z%）
学习时长：N 小时（目标 M 小时，达成率 P%）
连续学习：Q 天
测验：R 次，平均正确率 S%

═══ 掌握度变化 ═══
模块 A：65% → 78% (+13%) 📈
模块 B：40% → 35% (-5%) 📉
模块 C：0% → 25% (+25%) 📈（新开始）

═══ 亮点 ═══
• [具体成就和原因分析]

═══ 薄弱点 Top 3 ═══
1. [知识点] — 掌握度 X% — 主要错误类型 — 建议行动
2. ...
3. ...

═══ 错题分析 ═══
错误类型分布：概念不清 X% / 记忆模糊 Y% / 应用错误 Z% / 粗心 W%

═══ 效率评估 ═══
平均效率比：X（>1 学得快，<1 学得慢）
建议：[基于效率的调整建议]

═══ 下周计划调整 ═══
• [具体调整项和原因]
• [新增/跳过的节点]
• [复习安排]

═══ 行动清单 ═══
- [ ] 行动 1（优先级高）
- [ ] 行动 2
- [ ] 行动 3
```

**月报/阶段复盘格式（深度）：**
在周报基础上增加：
- 月度趋势图（掌握度曲线）
- 目标达成度评估
- 学习方法有效性分析
- 下月/下阶段规划建议
- 风险预警（如果当前节奏继续，能否按时完成目标）

### 第 4 步：同步到飞书

1. 尝试将复盘报告写入飞书文档
2. 更新飞书多维表格中的复盘记录
3. 通过消息卡片推送复盘摘要 + 文档链接

### 第 5 步：触发计划调整

复盘完成后，如果发现问题，自动调用 `learning-plan` 的動態調整规则：
- 掌握度持续下降 → 触发补强
- 效率持续下降 → 触发节奏调整
- 计划完成率持续 < 70% → 触发计划重排

## 注意事项

- **数据驱动**：所有结论必须有数据支撑，不说"感觉你学得不错"
- **具体可执行**：建议必须是具体行动，不说"继续努力"
- **对比基准**：和上周/上月对比，和计划对比，和目标对比
- **平衡视角**：既要指出问题，也要肯定进步
